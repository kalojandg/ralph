🎯 QUICK START - Стъпка по Стъпка:

════════════════════════════════════════════════════════════
ВАРИАНТ 1: Базово използване (най-просто)
════════════════════════════════════════════════════════════

   1️⃣ Създай PROMPT.md файл с твоята задача:
   
      # Task: Описание на задачата
      
      ## Requirements
      - Изискване 1
      - Изискване 2
      
      ## Success
      Когато завършиш, върни: <promise>COMPLETE</promise>

   2️⃣ Кликни два пъти на: ralph-quick.bat
   
   3️⃣ Готово! Ralph ще прави итерации докато:
      • Claude върне <promise>COMPLETE</promise>
      • Или достигне 10 итерации

════════════════════════════════════════════════════════════
ВАРИАНТ 2: С User-Defined стъпки (advanced)
════════════════════════════════════════════════════════════

   1️⃣ Създай PROMPT.md (основен PRD/план):
   
      # Task: Твоята главна задача
      <promise>COMPLETE</promise>

   2️⃣ Създай user-steps.md (специфични инструкции):
   
      # User Defined Steps
      
      Използвайки Playwright MCP:
      1. Отвори https://myapp.com
      2. Направи screenshot
      3. Провери дали виждаш X, Y, Z
      4. Ако ОК → <promise>COMPLETE</promise>

   3️⃣ Провери ralph-config.json:
   
      {
        "user_defined_steps": {
          "enabled": true,     ← ВАЖНО да е true!
          "steps_file": "user-steps.md"
        }
      }

   4️⃣ Кликни два пъти на: ralph-quick.bat
   
   5️⃣ В ВСЯКА итерация Claude вижда:
      • PROMPT.md (основната задача)
      • user-steps.md (твоите специфични стъпки)
      • feedback.md (feedback от предишна итерация)
      
      И изпълнява ВСИЧКО заедно!

════════════════════════════════════════════════════════════
ВАРИАНТ 3: С Feedback между итерации
════════════════════════════════════════════════════════════

   1️⃣ Създай PROMPT.md и (optional) user-steps.md
   
   2️⃣ Стартирай: ralph-quick.bat
   
   3️⃣ След няколко итерации, натисни Ctrl+C за пауза
   
   4️⃣ Редактирай feedback.md:
   
      # Feedback
      ## Забележки от досега:
      - Това работи добре
      - Това трябва да се поправи
      
      ## Фокус за следващите итерации:
      - Направи X
      - Поправи Y

   5️⃣ Стартирай отново - feedback се включва автоматично!

📝 МИНИМАЛНИ ИЗИСКВАНИЯ:

   ✓ Claude CLI инсталиран (провери: claude --version)
   ✓ PROMPT.md файл (създай го със задачата)
   ✓ ralph-config.json (вече съществува с defaults)
   
   Optional:
   • user-steps.md (за специфични инструкции)
   • feedback.md (за feedback loop)
   • completion-check.ps1 (за custom проверки)

💡 КАК РАБОТИ В ЕДНА ИТЕРАЦИЯ:

   ralph-iteration.ps1 прави:
   
   📖 Чете PROMPT.md          → Основна задача
   ➕ Append user-steps.md    → Твои специфични стъпки
   ➕ Append feedback.md      → Feedback от преди
   🤖 Изпраща ВСИЧКО на Claude
   💾 Записва log (ако enabled)
   ✅ Проверява за completion
   
   Ако не е готово → следваща итерация
   Ако е готово → спира ✓

🎯 ПРИМЕРИ:

   • PROMPT.md: "Implement user authentication"
   • user-steps.md: "Requirements: OAuth, 2FA, etc."
   • Ralph итерира докато всички features работят

   Bug fixing:
   • PROMPT.md: "Fix login bug"
   • completion-check.ps1: Runs tests
   • Ralph спира когато тестовете минават

❓ ПРОБЛЕМИ?

   • .bat файлът се затваря веднага:
     → Вече поправено! Пробвай отново.
   
   • Claude не се намира:
     → Провери: where claude
     → Инсталирай ако липсва

   • User steps не се добавят:
     → Провери ralph-config.json
     → "user_defined_steps.enabled" трябва да е true

   • Completion не работи:
     → Увери се че Claude връща <promise>COMPLETE</promise>
     → Или създай custom completion-check.ps1
